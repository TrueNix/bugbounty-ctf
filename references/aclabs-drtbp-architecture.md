# ACLabs DRTBP/Agenticverse Lab Architecture

## Overview

ACLabs labs are dynamically generated by the **DRTBP** (Don't Run The Bug, Play) / **Agenticverse** framework. Each lab instance is a self-contained Flask app with configurable vulnerabilities, variations, and defenses.

## Directory Structure

```
_lab/
├── configs/themes.json           # UI skins (corporate, darkweb, minimal, etc.)
├── src/
│   ├── target_app/
│   │   ├── app.py                # Main Flask factory (create_app)
│   │   └── page_renderer.py      # Themed HTML rendering
│   ├── vulnerabilities/          # One file per vulnerability class
│   │   ├── sqli_auth_bypass.py   # alpha
│   │   ├── open_redirect.py      # bravo
│   │   ├── broken_access_control.py  # charlie
│   │   └── ...
│   ├── defenses/                 # WAF, input filters, output filters
│   ├── kill_chain/               # CWE catalog, stage definitions
│   ├── skeleton_gen/             # Code generation for lab variants
│   └── orchestrator/             # Docker deployment, agent spawning
└── templates/                    # Dockerfiles, compose templates
```

## Variations

Each vulnerability class has multiple **variations** that change the attack technique:
- **default**: Classic string concatenation SQLi (SQLite)
- **blind**: Time-based blind SQLi (SLEEP/pg_sleep/randomblob)
- **union**: UNION-based injection with data reflection

Variations are selected via `DRTBP_VARIATION_CONFIG` env var (JSON).

## Defenses

Each variation declares which defenses are **applicable**:
- `WafEasyDefense`: Basic WAF pattern matching
- `WafHardDefense`: Advanced WAF with bypass detection
- `WafNonBypassableDefense`: Strict WAF (may block all SQLi)
- `ParameterizedGuardDefense`: Converts queries to parameterized
- `ErrorSuppressionDefense`: Hides SQL error messages

Defenses are applied via `DRTBP_DEFENSES` env var.

## DB Configuration

- `DB_TYPE`: sqlite | mysql | postgres (default: postgres)
- `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`: Connection params
- SQLite DBs are at `/tmp/drtbp_<codename>.db`

**Pitfall**: If `DB_TYPE=postgres` but no Postgres is running, the app falls through to SQLite but may not initialize tables. The DB file exists but is empty. Fix: manually create tables and seed users.

## Running Locally

```bash
cd _lab
PYTHONPATH=. python -m src.target_app.app
```

Port defaults to 5000 (via `TARGET_PORT` env var).

## Session Progress

- `/session/progress` — JSON with completed steps
- `/session/reset` — POST to reset progress
- Progress tracked by session cookie or `X-Session-Id` header
